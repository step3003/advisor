# Advisor — личный финансовый планировщик

Кроссплатформенное приложение для личного финансового учёта на **Go**: план трат
на месяц, учёт факта, сравнение план/факт, отчёты за произвольный период,
мультивалютность (базовая валюта — **BYN**, курсы НБ РБ).

Источник истины по требованиям — [`docs/ТЗ.md`](docs/ТЗ.md).

## Статус

Итерация 1 — **каркас + ядро** (домен + слой хранения). UI (Fyne) — следующая
итерация. Реализовано: доменный слой, порты, файловое хранилище-vault,
пересобираемый SQLite-индекс, клиент курсов НБ РБ, usecase-сервисы, сидинг
категорий, экспорт/импорт.

## Архитектура (Clean Architecture)

Зависимости направлены внутрь: `presentation → application → domain`;
`infrastructure` реализует порты. Домен не знает ни о Fyne, ни о SQLite, ни о
HTTP, ни о файловой системе.

```
cmd/advisor/                     — точка входа, ручная сборка зависимостей (DI)
internal/
├── domain/                      — СЛОЙ 1: сущности + бизнес-правила (только stdlib)
│   ├── core/                    —   общий kernel: Meta(rev/updated_at), EntryType, YearMonth, Date
│   ├── money/                   —   Money (int64 минорные единицы), пересчёт без float
│   ├── category/                —   Category/подкатегория, архивация
│   ├── transaction/             —   фактическая операция
│   ├── plan/                    —   MonthlyPlan / PlanItem
│   ├── recurring/               —   RecurringTemplate
│   └── report/                  —   value-объекты отчётов (PlanVsFact, PeriodSummary)
├── application/                 — СЛОЙ 2: usecase-сервисы
│   ├── ports/                   —   интерфейсы: репозитории, Vault, RateProvider/RateCache, Clock, IDGenerator
│   ├── catalog/                 —   управление категориями + сидинг (Приложение A)
│   ├── planning/                —   план, копирование из прошлого месяца, план vs факт
│   ├── ledger/                  —   CRUD транзакций, баланс месяца
│   ├── recurring/               —   идемпотентная генерация из шаблонов
│   ├── reporting/               —   агрегации за период, динамика по месяцам
│   ├── currency/                —   пересчёт в BYN по кэшу курсов
│   └── io/                      —   экспорт/импорт снапшота (JSON) + CSV
├── infrastructure/              — СЛОЙ 3: реализации портов
│   ├── vault/                   —   файловое хранилище (источник правды), конфликты iCloud
│   ├── index/sqlite/            —   локальный SQLite-индекс, миграции, RebuildFromVault, кэш курсов
│   ├── nbrb/                    —   HTTP-клиент курсов НБ РБ
│   ├── memory/                  —   in-memory адаптер портов (для тестов/прототипа)
│   ├── clock/                   —   системные часы
│   └── id/                      —   генератор UUID
└── presentation/                — СЛОЙ 4: Fyne UI (заглушка, следующая итерация)
```

### Модель хранения (ключевое)

- **Источник правды — vault**: папка с JSON-файлами по одному на запись
  (`<collection>/[YYYY-MM/]<uuid>.json`), модель как у Obsidian. Синхронизируется
  ОС (iCloud Drive).
- **SQLite-индекс — производный** и полностью пересобираемый из vault
  (`RebuildFromVault`). В vault не хранится (риск повреждения при облачной
  синхронизации целого файла). Кэш курсов НБ РБ — локальный, в индексе.
- **Деньги** — только `int64` минорные единицы и десятичные строки; никакого
  `float`.
- **Разрешение конфликтов iCloud**: конфликтные копии обнаруживаются, побеждает
  запись с большей `(rev, updated_at)`, проигравшая сохраняется в `_conflicts/`.

> Реальный iCloud-контейнер (entitlements, `iCloud.<bundle-id>`, файловый вотчер
> на iOS) подключается на этапе сборки iOS/macOS. Порт `Vault` спроектирован так,
> что путь к папке — параметр, и это не затрагивает домен/усечейсы.

## Сборка и запуск

Требуется Go 1.26. Драйвер SQLite — `modernc.org/sqlite` (чистый Go, без CGo).

```bash
go build ./...
go test ./...

# Инициализация хранилища и печать сводки (UI пока нет):
go run ./cmd/advisor -vault /path/to/Advisor -db /path/to/index.db
```

По умолчанию vault — `~/Advisor`, индекс — в каталоге конфигурации пользователя.
